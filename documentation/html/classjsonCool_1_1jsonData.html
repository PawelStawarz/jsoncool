Detailed Description

JSON data container.

Author
mgr inz. Pawel Stawarz
Date
05 november 2015

This class stores JSON structures of all possible types, supported by the JSON format. The class is used both for literal values (true, false and null ), strings, numbers, arrays and objects.

The value null is recognized as an object without any internal value. All other types of data have their own, appriopriate containers. One can retrieve the information about the data type by using the 'is*' function family.

null

To check whether the data is null , use the jsonData::isNull() function. When calling the jsonData::getNumericValue() function, null will be converted into '0'. When casting the object to a std::string, the result will be the string '"null"'. Trying to convert null into an array or object, will result in an exception being thrown. For exception handling consult jsonCool::jsonException

boolean values

To check whether the data is a boolean value, use the jsonData::isBoolean method. When calling the jsonData::getBooleanValue() function, standard JavaScript conversion rules apply:

  • null will be converted into false ,
  • true and false will have their appriopriate values,
  • A numeric value of 0 will yield false , and all other numeric values will be recognized as true ,
  • An empty string will yield false , and all other strigs will be recognized as true ,
  • An array is converted to true , no matter if, and what its members are,
  • An object is converted to true , no matter whether it has any parameters.

numeric values

To check whether the data is a numeric value, use the jsonData::isNumber method. When calling the jsonData::getNumericValue() function, standard JavaScript conversion rules apply:

  • null will be converted into 0 ,
  • false will be converted into 0 and true into 1 ,
  • A string containing only a number (for example '"20"'), will be converted into the appriopriate numeric value. An empty string will be converted into 0 and any other string will yield NaN ,
  • An empty array is converted into 0 . An array containing exactly one element is converted into the numeric value of that element. An array with more elements than one, yields NaN ,
  • An object is converted into NaN .

The NaN value is generated using the standard library nan() function, located within <cmath>. To test whether a value is NaN, use the isnan() function, from the standard C++ library.

strings

Since the whole idea behind JSON is for it, to be serializable, all values can be converted to a string. In order to convert data into a std::string, just use the overloaded conversion operator.

arrays

To check whether the data is an array, use the jsonData::isArray method. If the data is an array, jsonData::size() can be used to check its size. Iterating an array can be done by calling the overloaded jsonData::operator[](const std::vector<jsonData>::size_type index) . All array elements are of type jsonCool::jsonData , since JSON arrays are allowed to store values of mixed types. If you want to make a particular value into an empty array, use jsonCool::array (see the documentation of that object, for an example). You can add new elements into an existing array by calling the overloaded jsonData::operator()() .

objects

To check whether the data in an object, use the jsonData::isObject method. If the data is an object, jsonData::size() can be used to check its number of the object properties. In order to retrieve a list of names of the objects properties, call the jsonData::getMemberNames() function. The value of a particular property, can be retrieved by calling the overloaded jsonData::operator[](const std::string &parameter) or jsonData::operator[](const char* parameter) operators. You can add a new property by calling the above methods, and passing an unexisting property name as the function parameter.
